## `Utils` Functional Plan

| **类别**         | **模块名称 (英文)**          | **文件名 (File)**  | **模块前缀 (Prefix)** | **核心功能职责**                                                     |
| -------------- | ---------------------- | --------------- | ----------------- | -------------------------------------------------------------- |
| **I. 核心基础设施**  | **Config** (配置)        | `config.h/c`    | `cfg_`            | 加载和管理 API Key、数据库路径等全局配置。                                      |
|                | **Log** (日志/调试)        | `log.h/c`       | `log_`            | 提供统一的日志输出宏（如 `log_error`），包含错误级别和时间戳。                          |
| **II. C 语言增强** | **String Utils** (字符串) | `string.h/c`    | `str_`            | 封装 C 语言繁琐的字符串操作，如内存安全的复制、修剪 (`trim`)、分割 (`split`)。             |
|                | **Time Utils** (时间)    | `time_util.h/c` | `tim_`            | 处理 `time_t` 类型（Unix 时间戳）与人类可读格式（如 `YYYY-MM-DD`）之间的转换，供 CLI 显示。 |
| **III. 呈现优化**  | **Color** (颜色/终端)      | `color.h/c`     | `clr_`            | 实现 ANSI 颜色和终端格式化输出，供 `log` 和 `cli` 调用。                         |
