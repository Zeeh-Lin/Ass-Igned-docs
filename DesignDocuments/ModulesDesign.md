## Base Modules

| Name          | Module Prefix           | Description                                                    |
| ------------- | ----------------------- | -------------------------------------------------------------- |
| **utils**     | 按照不同功能，拥有不同前缀。例如：`dbg_` | **基础设施**。提供日志 (`LOG_INFO`, 字符串处理 (`trim`), 配置读取等底层工具，所有模块都依赖它。 |
| **database**  | `db_`                   | **数据持久层**。封装 SQLite3 操作，负责数据的“落地”。它是系统的“单一真理源”。                |
| **scheduler** | `sch_`                  | **核心逻辑层**。维护**最小堆 (Min-Heap)**，决定“下一个该做什么”。这是本项目的算法核心之一。       |
| **parser**    | `psr_`                  | **数据翻译层**。负责结构体与 JSON 互转，以及简单的用户指令解析（如 `add -t "work" ...`）。   |
| **cli**       | `cli_`                  | **交互层**。处理用户输入、调用逻辑层、并以 ANSI 彩色格式输出结果。                         |
| **ai_caller** | `aic_`                  | **智能连接器**。集成 `libcurl` 调用 DeepSeek API。                        |
